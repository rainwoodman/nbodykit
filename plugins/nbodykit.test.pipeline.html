nbodykit.test.pipeline module¶

class nbodykit.test.pipeline.RunAlgorithm(cmd, stdout=None, stderr=None, poll_time=0.01, timeout=None)¶

Bases: pytest_pipeline.core.PipelineRun

Run a pipeline that tests an Algorithm. Prior to running this class will:

  • load the necessary data to the cache directory
  • initialize the output directory in nbodykit/examples
  • set the NBKIT_CACHE and NBKIT_HOME directories

nbodykit.test.pipeline.add_run_fixture(modname, basecls, name, make_fixture=<function default_run_fixture>, timeout=60)¶

A class decorator that creates a run fixture and adds it to the class as a pytest fixture

Parameters:

modname : str

the name of the module to which the fixture class should be attached; this is needed so that each test module knows where to find the relevant fixtures

basecls : subclass of PipelineRun

the base class for the run fixture, with any functions added that will be executed prior to the pipeline run

name : str

the name of the nbodykit Algorithm to run

make_fixture : callable, optional

a function that returns a class fixture that runs the pipeline; default is default_run_fixture

timeout : int, optional

an optional timeout in seconds – if this limit is exceeded, the analysis run is terminated and the test fails; default is 60 seconds
nbodykit.test.pipeline.default_run_fixture(cls, name, timeout=60)¶

Create a class fixture to run the analysis pipeline

Parameters:

cls : subclass of PipelineRun

a subclass of PipelineRun that will create the run fixture

name : str

the name of the nbodykit Algorithm to run

timeout : int, optional

an optional timeout in seconds – if this limit is exceeded, the analysis run is terminated and the test fails

Notes

  • The input cls should have attributes param_file which specifies the input configuration file and run_dir, which specifies where param_file lives
  • The analysis run will be performed in parallel using 2 processes
